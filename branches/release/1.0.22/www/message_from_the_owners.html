Scarab 1.0.22 Release Candidate 1 is available.

The Scarab team is pleased to announce the first release candidate of Scarab 1.0.22. It‘s the upcoming replacement for the current stable 1.0.21 release, bringing a bunch of new features and many enhancements and fixes.

Note: This is a NOT a final release. As such, do not install it in production environments - even if it‘s already extensively tested and considered stable. The final stable version will be released shortly.

In addition to many smaller enhancements and fixes the new release offers:

  • Modifiable display attributes for dependant issues.
  • Report engine date ranges support
  • Report can be defined for module home page.
  • Major rework of the history tab. Fixes for date formats, layout and more…
  • Enhanced automatic login by NTLM.
  • Property for maximum file size of uploads: scarab.file.max.size
  • Enhanced full text search engine.
  • Support for multi-part MIME email for sending plain-text-with-HTML email.
  • Notifications by email work threaded now
  • Handler to create new tickets and to handle attribute options via xml-rpc interface.
  • Reworked screen for role requests.
  • Added permission for role requests.
  • Now uses Torque 3.3 and Velocity 1.6

0.21 is still the current stable release of Scarab, and our recommended version for production systems.
